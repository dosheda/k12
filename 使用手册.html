1. 这是什么？

K12 古诗词讲解助手是一个网页版 AI 工具，专门帮助小学生和初中生学习古诗。

你可以把它想象成一位住在电脑里的语文家教老师——我们叫她 「小K老师」。

🎯 核心思路

孩子问一个问题（比如「有没有描写春天的诗？」），小K老师从知识库里翻出最相关的几首诗，然后用 AI 针对这些诗进行讲解——而不是凭空编造。这就叫 RAG（检索增强生成）。

简单说：AI 只能讲知识库里有的诗，不会瞎编。

知识库里有什么？

收录了从汉代到清代的 80 首经典古诗词，涵盖小学到初中常见的必读篇目。每首诗都包含：

  • 原文 — 诗的完整文字
  • 赏析 — 这首诗好在哪里、用了什么手法
  • 四维标签 — 题材类型、表达情感、表现手法、关键意象（共 4 个维度，方便精准检索）

2. 怎么启动？

1

打开终端（PowerShell）

按键盘上的 Win + R，输入 powershell，回车。

2

设置 DeepSeek API Key 和访问口令（仅第一次需要）

在终端里粘贴下面两行（把占位内容换成你自己的）：

$env:DEEPSEEK_API_KEY = "你的 DeepSeek API Key"
$env:K12_HELPER_ACCESS_CODE = Read-Host "请输入网页访问口令"
💡 这个 Key 去哪里弄？去 DeepSeek 开放平台 注册一个账号，在「API Keys」页面创建一个就有了。
3

启动程序

先进入项目文件夹，再启动：

cd "你的项目文件夹"
python -m streamlit run app.py

稍等几秒（首次启动要加载模型），终端里出现一行地址后就成功了。

4

打开浏览器

在浏览器地址栏输入：

http://localhost:8501

第一次进入需要输入访问口令；勾选“在此设备记住 7 天”后，同一浏览器刷新页面不用反复输入。看到页面就可以开始用了！🎉

⚠️ 注意：终端（PowerShell 窗口）不能关掉，关掉程序就停了。不用的时候可以按 Ctrl + C 退出。
💡 小技巧：不要把 API Key 写进启动脚本。建议把 DEEPSEEK_API_KEY 和 K12_HELPER_ACCESS_CODE 设置为系统用户环境变量；启动脚本里只保留 python -m streamlit run app.py 这一类不含密钥的命令。
🔐 访问口令：访问口令只用于进入网页，不是 DeepSeek API Key。登录成功后当前浏览器会记住 7 天；如果是在公用设备上使用，可以取消勾选“在此设备记住 7 天”，或在侧边栏点击“退出登录”清除记住状态。

4. 功能介绍

🧠 功能一：智能古诗讲解（核心功能）

在底部输入框输入问题，按下回车，小K老师就会回答。每次回答都包括：

  • 【原文】 — 诗的标题、作者、完整原文
  • 【这首诗讲了什么】 — 用孩子能听懂的话概括内容，营造画面感
  • 【作者想表达什么】 — 分析诗人的情感和写作动机
  • 【写得好的地方】 — 讲解这首诗的妙处（基于知识库中的赏析）
  • 【小知识】 — 和诗或诗人有关的趣味小故事（可选）
试试这样问：
  • 和友情有关的诗有哪些？
  • 望庐山瀑布好在哪里？
  • 有没有描写春天的诗？
  • 李白写过哪些写月亮的诗？
  • 再讲一首（配合多轮对话使用）

🔄 功能二：多轮对话

小K老师能记住前面聊过的内容。比如：

  1. 先问：「望庐山瀑布好在哪里？」→ 小K老师讲解
  2. 再问：「它的作者是谁？」→ 小K老师知道「它」指的是上一首
  3. 再问：「再讲一首」→ 小K老师从上次的检索结果里再挑一首

最多保留 5 轮对话，超出后自动丢弃最早的记录。

🗣️ 功能三：语音朗读

每条老师回答的下面都有两个按钮：

  • 🔊 朗读 — 用浏览器自带的语音引擎把回答念出来（中文发音，稍慢语速）
  • ⏹ 停止 — 随时停止朗读
💡 语音朗读用的是浏览器内置的 TTS（文字转语音）功能，不需要任何 API Key，完全免费。所有现代浏览器（Chrome、Edge、Firefox、Safari）都支持。

📊 功能四：学情分析

这是新增的智能功能，帮助家长了解孩子的学习情况。

自动记录

每次孩子提问时，系统会自动记录：

  • 问了什么问题
  • 真正展开讲解或复习了哪些诗
  • 只是点名但没有展开的诗
  • 本轮检索到的候选诗（只作存档，不算已学习）
  • 真正学习/复习过的诗涉及哪些主题标签（如思乡、写景、送别……）
  • 提问的时间

数据存在本地 SQLite 数据库里，关掉浏览器下次打开还在。

实时统计

左侧边栏「📈 学习概览」实时显示：总提问次数、真正学习了多少首不同的诗、复习次数、仅提及/未命中次数、最常练习主题和已覆盖主题。

生成学情报告

点击侧边栏「📊 生成学情报告」按钮，AI 会分析孩子的学习数据，并以弹窗方式生成一份给家长看的分板报告，包含 5 个板块：

板块内容
📊 学习概览提问次数、真正学习诗数、复习次数
🔍 兴趣分析孩子对哪些主题最感兴趣
⚖️ 均衡性评估学习覆盖面是否均衡、缺少哪些类型
💡 拓展建议推荐还没真正学习过的诗歌类型
🌟 学习寄语温暖的鼓励和后续学习方向
💡 数据不足时的处理：如果只问了 1-2 个问题，系统不会调用 AI，而是直接提示「学习记录还比较少，多用一会儿就能生成更准确的分析」——省 API 费用。

🔍 混合检索是怎么工作的？

这是整个系统最核心的设计——两路检索同时跑，结果合并：

第一路：语义向量检索（传统的 RAG 做法）

  1. 把孩子的问题（如「有没有描写春天的诗」）用 bge 模型转成一个 384 维的数学向量
  2. 拿这个向量去 Chroma 里找最相似的 15 首诗（按向量距离排序）
  3. 优点：能理解近义词，搜「春天」也能找到写「春晓」的诗

第二路：关键词 / 标签匹配（新增的第二路）

  1. 把孩子的问题拆成 2字/3字/4字 n-gram（如「描写」「春天」「描写春天」）
  2. 每个 n-gram 去每首诗的全文 + 标签里做子串匹配
  3. 命中的关键词越长，加分越多（4字词 > 3字词 > 2字词）
  4. 标签命中（如「写景、春天、生机」）直接拉高这首诗的排名

合并规则

  • 语义路径的得分：1.0 − 归一化距离（0~1 之间）
  • 关键词路径的得分：归一化匹配分 × 2.0（标签命中加权更高）
  • 同一首诗两路都命中 → 得分叠加（排名飙升）
  • 只在关键词路命中的诗也收进来（不丢掉）
  • 最终按得分从高到低排序，取 前 8 首 交给 DeepSeek

为什么要做混合检索？因为纯向量检索有时候搜不准——比如问「有没有写友情的诗」，向量检索可能搜到一些聊天内容相关的诗，但标签里明确有「友情、送别」的诗反而没排到前面。加上标签匹配后，这类问题就准多了。

6. 常见问题

Q: 打开网页后显示「找不到向量库文件夹」？

A: 说明项目目录下找不到 chroma_db 文件夹。请确认文件夹位置正确，或者重新运行一次 build_rag_db.py。

Q: 回答问题时报错「API Key 无效」？

A: 检查两件事：① 环境变量 DEEPSEEK_API_KEY 设置了没有（在终端输入 $env:DEEPSEEK_API_KEY 看返回值）；② Key 是不是从 DeepSeek 开放平台复制的完整版本。局域网访问还需要设置 K12_HELPER_ACCESS_CODE。

Q: 回答很慢？

A: 每次提问需要等几秒到十几秒，因为 DeepSeek 需要时间思考并生成回答。属于正常现象。如果超过 30 秒没反应，可能是网络问题。

Q: 小K老师讲的诗是知识库里没有的？

A: 不应该。系统 prompt 明确要求 AI「只能依据提供的诗来回答，不要编造」。如果出现了这种情况，可能是 AI 偶尔不遵守规则，属于大模型的通病，概率很低。

Q: 学情报告的数据存哪里了？会不会泄露？

A: 学习记录存在本地 learning_records.db 文件里（SQLite 数据库）。普通提问和生成学情报告时，问题和必要摘要会发送给 DeepSeek API 生成回答；如果以后输入真实学生数据，请先确认隐私和数据保留要求。

Q: 怎么清空学习记录重新开始？

A: 退出程序后，删除项目目录下的 learning_records.db 文件即可。下次提问时会自动创建一个新的空数据库。

Q: API 费用贵不贵？

A: DeepSeek V4 Pro 的 API 价格很低（几毛钱就能跑很多次）。学情报告在数据不足 3 条时不会调用 API。只提问不讲诗的话，一次大概消耗几千 token，成本极低。

Q: 能在手机上用吗？

A: 可以！在电脑上启动后，手机连同一个 WiFi，浏览器访问 http://电脑的IP地址:8501（启动时终端里显示的「Network URL」就是）。
