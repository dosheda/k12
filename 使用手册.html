1. 这是什么？

K12 古诗词讲解助手是一个网页版 AI 工具，专门帮助小学生和初中生学习古诗。

你可以把它想象成一位住在电脑里的语文家教老师——我们叫她 「小K老师」 👩‍🏫。

🎯 核心思路

孩子问一个问题（比如「有没有描写春天的诗？」），小K老师从知识库里翻出最相关的几首诗（并把这些来源诗亮在回答上方），再用 AI 针对这些诗进行讲解——而不是凭空编造。这就叫 RAG（检索增强生成）。

简单说：AI 只能讲知识库里有的诗，不会瞎编；而且讲了哪几首，你一眼看得见。

知识库里有什么？

收录了从汉代到清代的 80 首经典古诗词，涵盖小学到初中常见的必读篇目。每首诗都包含：

  • 原文 — 诗的完整文字
  • 赏析 — 这首诗好在哪里、用了什么手法
  • 四维标签 — 题材类型、表达情感、表现手法、关键意象（共 4 个维度，方便精准检索）

2. 怎么启动？

1

打开终端（PowerShell）

按键盘上的 Win + R，输入 powershell，回车。

2

设置 DeepSeek API Key 和访问口令（仅第一次需要）

在终端里粘贴下面两行（把占位内容换成你自己的）：

$env:DEEPSEEK_API_KEY = "你的 DeepSeek API Key"
$env:K12_HELPER_ACCESS_CODE = Read-Host "请输入网页访问口令"
💡 这个 Key 去哪里弄？去 DeepSeek 开放平台 注册一个账号，在「API Keys」页面创建一个就有了。
3

启动程序

先进入项目文件夹，再启动：

cd "你的项目文件夹"
python -m streamlit run app.py

稍等几秒（首次启动要加载模型），终端里出现一行地址后就成功了。

4

打开浏览器

在浏览器地址栏输入：

http://localhost:8501

第一次进入需要输入访问口令；勾选“在此设备记住 7 天”后，同一浏览器刷新页面不用反复输入。看到页面就可以开始用了！🎉

⚠️ 注意：终端（PowerShell 窗口）不能关掉，关掉程序就停了。不用的时候可以按 Ctrl + C 退出。
💡 小技巧：不要把 API Key 写进启动脚本。建议把 DEEPSEEK_API_KEY 和 K12_HELPER_ACCESS_CODE 设置为系统用户环境变量；启动脚本里只保留 python -m streamlit run app.py 这一类不含密钥的命令。
🔐 访问口令：访问口令只用于进入网页，不是 DeepSeek API Key。登录成功后当前浏览器会记住 7 天；如果是在公用设备上使用，可以取消勾选“在此设备记住 7 天”，或在侧边栏点击“退出登录”清除记住状态。

4. 功能介绍

🧠 功能一：智能古诗讲解（核心功能）

在底部输入框输入问题，按下回车，小K老师就会回答。回答上方会先亮出检索到的来源诗卡片，正文再逐字流式呈现，包含：

  • 【原文】 — 诗的标题、作者、完整原文
  • 【这首诗讲了什么】 — 用孩子能听懂的话概括内容，营造画面感
  • 【作者想表达什么】 — 分析诗人的情感和写作动机
  • 【写得好的地方】 — 讲解这首诗的妙处（优先转述知识库中的赏析）
  • 【小知识】 — 和诗或诗人有关的趣味小故事（可选）
试试这样问：
  • 和友情有关的诗有哪些？
  • 望庐山瀑布好在哪里？
  • 有没有描写春天的诗？
  • 李白写过哪些写月亮的诗？
  • 再讲一首（配合多轮对话使用）
💡 来源可见可点：回答上方卡片就是这次真正检索到的诗。点任意一张卡片的诗名，会打开「古诗库」并定位到这首诗，直接看整理全文。

🔄 功能二：多轮对话

小K老师能记住前面聊过的内容。比如：

  1. 先问：「望庐山瀑布好在哪里？」→ 小K老师讲解
  2. 再问：「它的作者是谁？」→ 小K老师知道「它」指的是上一首
  3. 再问：「再讲一首」→ 小K老师从上次的检索结果里再挑一首

最多保留 5 轮对话，超出后自动丢弃最早的记录。聊天正文会按浏览器会话保存，刷新或重开浏览器可恢复最近 5 轮。

🗣️ 功能三：语音朗读

每条老师回答的下面都有两个胶囊按钮：

  • 🔊 朗读 — 用浏览器自带的语音引擎把回答念出来（中文发音，稍慢语速）
  • ⏹ 停止 — 随时停止朗读
💡 语音朗读用的是浏览器内置的 TTS（文字转语音），不需要任何 API Key，完全免费。所有现代浏览器（Chrome、Edge、Firefox、Safari）都支持。

📊 功能四：学情分析

帮助家长了解孩子的学习情况。

自动记录（分得很细）

每次孩子提问时，系统会自动区分并记录：

  • 问了什么问题、提问时间
  • 真正展开讲解的诗（算作新学习）
  • 复习：已学过、这次又展开讲的诗
  • 仅提及：只是点名对比、没有展开的诗（不计入已学习）
  • 本轮检索到的候选诗（只作存档，不算已学习）
  • 真正学习/复习过的诗涉及哪些主题标签（思乡、写景、送别……）

数据存在本地 SQLite 数据库里，关掉浏览器下次打开还在。

可视化实时统计

左侧「📈 学习概览」用指标卡 + 进度条 + 彩色条形图实时展示：总提问次数、真正学习了多少首不同的诗、已学进度（X / 80）、复习/仅提及/未命中次数，以及最常练习主题、已覆盖主题。

多用户各自独立

💡 学情按学习者区分：同一浏览器/设备始终是同一个孩子，统计长期累积；不同设备或浏览器互相隔离，同一局域网多个孩子不会共用一份统计。点「开始新对话」只换聊天、不会清零学情。

生成学情报告（弹窗内就地生成）

点击侧边栏「📊 生成学情报告」，弹窗会立刻打开并显示生成动画（不再对着页面干等），AI 分析完成后自动替换成报告正文，含 5 个板块：

板块内容
📊 学习概览提问次数、真正学习诗数、复习次数
🔍 兴趣分析孩子对哪些主题最感兴趣
⚖️ 均衡性评估学习覆盖面是否均衡、缺少哪些类型
💡 拓展建议推荐还没真正学习过的诗歌类型
🌟 学习寄语温暖的鼓励和后续学习方向
💡 数据不足时的处理：如果只问了 1-2 个问题，系统不会调用 AI，而是直接提示「学习记录还比较少」——省 API 费用。
💡 查看 / 重新生成：生成过报告后，侧边栏会出现「📄 查看上次报告」和「🔄 重新生成学情报告」；再看一眼用前者（不耗时），需要基于最新记录重新分析时用后者。

📚 功能五：古诗库浏览

点侧边栏「📚 古诗库（浏览全部）」，弹窗里可以主动逛遍全部 80 首诗：

  • 按诗名 / 诗句 / 标签关键词搜索
  • 按标签下拉筛选（如只看「送别」类）
  • 展开任意一首，查看标签与整理全文（含赏析）
💡 回答上方来源卡片的诗名也能点，会直接打开古诗库并定位到那首诗。

🔍 混合检索是怎么工作的？

这是整个系统最核心的设计——两路检索同时跑，结果合并：

第一路：语义向量检索（传统的 RAG 做法）

  1. 把孩子的问题用 bge 模型转成数学向量
  2. 拿这个向量去 Chroma 里找最相似的 15 首诗（按向量距离排序）
  3. 优点：能理解近义词，搜「春天」也能找到写「春晓」的诗

第二路：关键词 / 标签匹配（新增的第二路）

  1. 把孩子的问题拆成 2字/3字/4字 n-gram（如「描写」「春天」「描写春天」）
  2. 每个 n-gram 去每首诗的全文 + 标签里做子串匹配
  3. 命中的关键词越长，加分越多（4字词 > 3字词 > 2字词）
  4. 标签命中（如「写景、春天、生机」）直接拉高这首诗的排名

合并规则

  • 语义路径的得分：1.0 − 归一化距离（0~1 之间）
  • 关键词路径的得分：归一化匹配分 × 2.0（标签命中加权更高）
  • 同一首诗两路都命中 → 得分叠加（排名飙升）
  • 只在关键词路命中的诗也收进来（不丢掉）
  • 最终按得分从高到低排序，取 前 8 首 交给 DeepSeek

为什么要做混合检索？因为纯向量检索有时候搜不准——比如问「有没有写友情的诗」，标签里明确有「友情、送别」的诗反而没排到前面。加上标签匹配后，这类问题就准多了。

6. 常见问题

Q: 打开网页后显示「找不到向量库文件夹」？

A: 说明项目目录下找不到 chroma_db 文件夹。请确认文件夹位置正确，或者重新运行一次 build_rag_db.py。

Q: 回答问题时报错「API Key 无效」？

A: 检查两件事：① 环境变量 DEEPSEEK_API_KEY 设置了没有（在终端输入 $env:DEEPSEEK_API_KEY 看返回值）；② Key 是不是从 DeepSeek 开放平台复制的完整版本。局域网访问还需要设置 K12_HELPER_ACCESS_CODE。

Q: 点了提问，来源卡片出来后好像卡住了？

A: 那是 AI 在准备第一个字。这段时间会显示「小K老师正在动笔讲解…✍️」的动画提示，之后回答会逐字流式出现，属于正常现象。如果超过 30 秒没反应，可能是网络问题。

Q: 小K老师讲的诗是知识库里没有的？

A: 不应该。系统 prompt 明确要求 AI「只能依据提供的诗来回答，不要编造」，而且检索到的来源诗就展示在回答上方，可以对照。偶尔不遵守规则属于大模型通病，概率很低。

Q: 家里有两个孩子，学情会混在一起吗？

A: 不会。学情按学习者区分：不同浏览器/设备各自独立累积。若想让两个孩子分开统计，让他们用不同的浏览器或设备访问即可。

Q: 学情报告的数据存哪里了？会不会泄露？

A: 学习记录与聊天正文存在本地 learning_records.db（SQLite）。普通提问和生成学情报告时，问题和必要摘要会发送给 DeepSeek API 生成回答；如果以后输入真实学生数据，请先确认隐私和数据保留要求。

Q: 怎么清空学习记录重新开始？

A: 退出程序后，删除项目目录下的 learning_records.db 文件即可。下次提问时会自动创建一个新的空数据库。

Q: API 费用贵不贵？

A: DeepSeek V4 Pro 的 API 价格很低（几毛钱就能跑很多次）。学情报告在数据不足 3 条时不会调用 API。只提问不讲诗的话，一次大概消耗几千 token，成本极低。

Q: 能在手机上用吗？

A: 可以！在电脑上启动后，手机连同一个 WiFi，浏览器访问 http://电脑的IP地址:8501（启动时终端里显示的「Network URL」就是）。
