## 项目概述
- 这个软件解决的问题：给 K12 学习场景提供本地运行的古诗词讲解助手，用户提问后从本地古诗词知识库检索相关内容，再调用 DeepSeek 生成适合学生理解的讲解。
- 核心功能：
  - Streamlit 网页聊天入口：`app.py`。
  - 本地 Chroma 向量库检索 80 首古诗词：`chroma_db/`。
  - DeepSeek API 生成古诗词讲解和学情报告。
  - SQLite 本地记录学习行为：`learning_records.db`，当前为模拟数据。
  - 命令行 RAG/检索脚本：`rag_chat.py`、`search_rag.py`。
  - 命令行错题讲解与 OCR 识别脚本：`k12_helper.py`。
  - 古诗文本合并、整理、标签生成、Chroma metadata 更新脚本。
- 明确不做的事：暂无已确认的“不做”事项。以后用户明确说“这个不做”时再补充。

---

## 技术栈
- 语言/框架：Python + Streamlit。
  - 为什么选它：当前代码已经用 Python 完成数据处理、SQLite、Chroma 和 DeepSeek 调用；Streamlit 能快速提供本地/局域网页面，适合这个单机学习助手阶段。
- 模型/API：DeepSeek API，通过 OpenAI SDK 兼容接口调用。
  - API key 必须来自系统环境变量 `DEEPSEEK_API_KEY`，不要写入代码、`.bat`、文档示例或可提交文件。
- 向量数据库：ChromaDB，本项目当前只允许本地 `PersistentClient` 方式，直接读写本地 `chroma_db/`。
  - 已确认：不以 Chroma HTTP server 方式运行，不开放 Chroma 网络端口。
- Embedding：`BAAI/bge-small-zh-v1.5`，通过 Chroma embedding function 加载。
- 数据库：SQLite，本地保存学习记录。
- OCR：Tesseract + Pillow，用于命令行图片题识别。
- 依赖管理：当前 `requirements.txt` 未锁版本，这是待修问题；在修复前不要随意升级或新增重型依赖。
- 约定：未经用户同意，不要更换框架，不要引入新的重型依赖，不要把本地 Chroma 改成 HTTP server 架构。

- 当前根目录混有源码、脚本、数据和运行时文件；整理结构前先征得用户同意。

## 安全红线（直接生效，防止以后改回去）
- 严禁把 API key、密码、token 等密钥硬编码进代码，一律用系统环境变量或受控密钥配置。
- 严禁把 API key 写进 `.bat`、`.env`、说明文档的真实示例、截图或任何可能被提交/分享的文件。
- 初始化 git 或提交文件前，必须先有 `.gitignore`，至少排除 `.env`、密钥文件、`*.db`、`learning_records.db`、`chroma_db/`、`__pycache__/`、真实启动脚本和凭证文件。
- 当前 Chroma 只允许本地 `PersistentClient` 读写 `chroma_db/`；不得改成 Chroma HTTP server，不得开放 Chroma 网络端口，除非先完成依赖升级、鉴权设计和安全审查，并得到用户确认。
- 项目已确认会给同一局域网用户访问；任何局域网可访问的功能都不能默认裸奔。上线给他人使用前，必须处理访问控制、API 调用限流和输入长度限制。
- 所有外部输入（聊天输入、API 参数、文件内容、图片路径、OCR 文件）都要校验，特别是长度、文件大小、文件类型和路径边界。
- 数据库操作必须防注入：使用参数化查询，不要拼接 SQL 字符串。
- 错误信息不要把堆栈、系统路径、API 响应原文、密钥、token 等敏感信息暴露给最终用户。
- 学习记录当前是模拟数据；如果后续接入真实学生/家长数据，必须补隐私提示、第三方 API 数据流说明和数据清空/保留策略。
- 会删除、覆盖、重建数据的脚本必须有备份、明确确认或 `--force` 机制；不得静默删除 `chroma_db/` 或覆盖古诗词数据文件。
- DeepSeek API 调用涉及成本，任何会增加 API 调用量的功能都要考虑频率限制、输入长度限制和失败重试策略。
